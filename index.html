Designed for the end-user, administrator & developer

Need reasons to use this plugin? Check the features below.


Quick.

Installation is as easy as eating pie. Simply download the plugin from Bukkit, move the .jar to the plugins directory of your server, assign permissions (or OP!) and restart the server.

Persistent.

Identity Crisis saves all information to the configuration file (config.yml) which allows name changes to remain cross-restart/reload.

Lightweight.

Ensuring high amounts of memory aren't used, Identity Crisis uses simple methods to keep new names loaded in memory. This ensures that your server runs optimally.

Open-Source.

As Identity Crisis is Open-Source, anyone is able to submit bug-fixes or new features to the repo allowing for faster fixes and a tighter community integration.

Easy.

With the small amount of permissions and commands, using Identity Crisis couldn't be easier! More information about usage can be found on the "Usage" tab.

Tag-API.

With only one plugin dependency, Tag-API is a good one at that. Developed by mbaxter and always up to date, Tag-API allows me to change the usernames (or 'tags') above players heads to allow the full crisis.

Commands


/changename

Command parameters: <old name> <new name>
Notes: The user doesn't have to be online, thus, the old name must be exact. If the user isn't online, their name change will be assigned on their next join. Colors can also be used in the command with &col-code; styles must be applied before colors. List of colors available here.


/resetname

Command parameters: <old name>
Notes: The user doesn't have to be online, thus, the old name must be exact. If the user isn't online, their login-name will be their name throughout the server.

Permissions


identitycrisis.use

This permission node allows use to all commands. There are no nodes that restrict the viewing of nicknames. Operators have access to this node by default.

Developer API

The plugin has a developer API which allows you to change players usernames through Bukkit plugins. You are easily able to see if a player has a changed username, get the changed username, change the username, remove the username change and see if they have joined during the session.

Checking if username is changed

IdentityCrisis ic = (IdentityCrisis) this.getServer().getPluginManager().getPlugin("IdentityCrisis");
boolean nameChanged = ic.getName("jamietech") != null;

Getting changed username

IdentityCrisis ic = (IdentityCrisis) this.getServer().getPluginManager().getPlugin("IdentityCrisis");
String newName = ChatColor.translateAlternateColorCodes('&', ic.getName("ammaraskar"));

Changing username

Using IdentityCrisis.chopString(String); 'chops' the string supplied to be 16 characters in length to ensure that the name isn't too long for Minecraft.

IdentityCrisis ic = (IdentityCrisis) this.getServer().getPluginManager().getPlugin("IdentityCrisis");
try {
    ic.addNameChange("I_am_not_funny", ic.chopString(ChatColor.RED + "I_am_not_funny"));
} catch (TooBigException ex) {
    // Ignore exception as it won't be fired.
}

Removing username change

IdentityCrisis ic = (IdentityCrisis) this.getServer().getPluginManager().getPlugin("IdentityCrisis");
ic.removeNameChange("mbaxter");

Checking if user has joined this session

IdentityCrisis ic = (IdentityCrisis) this.getServer().getPluginManager().getPlugin("IdentityCrisis");
boolean hasJoined = ic.contains("isybub");
